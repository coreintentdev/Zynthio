## Release Cadence Strategy

| Cadence | Recommended For | Notes |
|---------|----------------|-------|
| Weekly single | Growth phase | Aggressive — only if catalogue depth allows |
| Bi-weekly single | Steady build | Good for algorithmic momentum without burnout |
| Monthly single | Sustainable long-term | Recommended starting point |
| EP (4–6 tracks) | Every 3–6 months | Bundle singles + 1–2 new exclusives |
| Album (8–12 tracks) | Annually | Only when you have the catalogue |

**Recommendation for DJ Zynrose:** Start with **monthly singles** to build Spotify algorithmic momentum, then compile into EPs every quarter.

## Per-Release Timeline Template

Use this timeline for every single or EP. Work backwards from your target release date.

```
T-28 days  Finalise master WAV — send to mastering engineer (if external)
T-21 days  Receive mastered WAV — quality check
T-21 days  Cover art finalised (3000×3000 px)
T-21 days  Metadata locked (title, genre, release date, explicit flag)
T-21 days  Lyrics file complete → lyrics/track-slug.txt
T-21 days  Submit to DistroKid (minimum 7 days before release; 3 weeks for editorial pitch window)
```
