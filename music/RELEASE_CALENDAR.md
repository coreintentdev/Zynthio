## Release Campaign Block Template

Copy and fill in one block per release:

```markdown
---

### Release: [RELEASE TITLE]

**Type:** Single / EP / Album
**Target Release Date:** YYYY-MM-DD
**DistroKid Submission Deadline:** YYYY-MM-DD (at least 14 days before release)
**Genre:** [e.g. Electronic / House / DnB]
**Tracks:** [list track names]

#### Milestone Checklist

**Production**
- [ ] WAV master(s) finalised
- [ ] Cover art designed (3000×3000px)
- [ ] Lyrics written and saved to lyrics/[slug].txt
- [ ] Metadata sheet completed

**Distribution (8 weeks out — YYYY-MM-DD)**
- [ ] DistroKid submission complete
- [ ] ISRC(s) noted in TRACK_MANIFEST.md
- [ ] UPC noted in TRACK_MANIFEST.md
- [ ] HyperFollow link created and tested

**Playlist Pitching (6–8 weeks out — YYYY-MM-DD)**
- [ ] Spotify for Artists editorial pitch submitted
- [ ] SubmitHub / Groover submissions sent (budget: $XX)
- [ ] Press release drafted

**Social Media Setup (4 weeks out — YYYY-MM-DD)**
- [ ] Content calendar drafted (see MUSIC_MARKETING.md)
- [ ] Announcement post scheduled
- [ ] Spotify Canvas created and uploaded
- [ ] TikTok/Reels clip cut and ready

**Release Week**
- [ ] Release live on all stores (verify)
- [ ] Release day post published on all platforms
- [ ] HyperFollow link updated to stream links
- [ ] YouTube visualiser / audio video live

**Post-Release (weeks 1–4)**
- [ ] Monitor streams and playlist adds daily
- [ ] Engage with comments and shares
- [ ] Share any playlist additions on Stories/posts
- [ ] TRACK_MANIFEST.md status updated to `released`
- [ ] Revenue dashboard checked at 30 days
- [ ] APRA AMCOS registration submitted (NZ)

**Notes:**
[Add any release-specific notes here]

---
```
